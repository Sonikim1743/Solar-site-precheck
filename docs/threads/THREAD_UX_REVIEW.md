# Thread Guide: UX / Practical Review

## Purpose

This thread reviews the app and portal from the actual worker's perspective.

The goal is not to make the UI flashy. The goal is to make it easy to complete work.

## Read first

1. `docs/project/PROJECT_CONTEXT.md`
2. `docs/project/PROJECT_WORKFLOW.md`
3. `docs/threads/THREAD_UX_REVIEW.md`

## Review philosophy

The user prefers:

- Clear action buttons.
- Less text.
- Bigger readable fonts.
- Layouts that do not overlap on mobile.
- Maps and results that are visually connected.
- Practical Japanese labels.
- Hints only where they reduce confusion.

The user dislikes:

- Excessive explanation.
- Strategy tiles that do not do anything.
- Request cards that distract from current functions.
- Floating buttons overlapping content.
- UI elements that feel decorative rather than useful.

## Review targets

### App

- Map search.
- Candidate info display.
- NEDO snow section.
- Horizon analysis.
- Terrain profile.
- Solar Pro CSV output.
- Report view.
- Mobile Safari behavior.

### Portal

- Hero clarity.
- Tool section usefulness.
- Module database card readability.
- Guide card density.
- Cases and Regions usefulness.
- Policy section length.
