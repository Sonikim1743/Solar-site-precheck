# Solar Site Precheck

Solar Proへ入力する前に、太陽光発電候補地の位置・標高・周辺地形・積雪リスク・現地メモをまとめるためのReact製MVPです。発電量の計算やSolar Pro本体との直接連携は行いません。

## MVPでできること

- 国土地理院の航空写真をLeafletで表示（標準地図へ切替可能）
- 地図クリックによる緯度・経度の取得と度・分表示
- 法務省の登記所備付地図XML・ZIPをアプリ内で変換し、航空写真へ重ねて地番検索・筆選択（GeoJSONにも対応）
- 国土地理院の住所検索を使った住所・地名検索
- 国土地理院DEMから補助標高を取得し、NEDO PDF読込時は帳票標高を優先
- 250 m〜5 kmの標高から地平線仰角を概算（通常は8方位、詳細分析チェック時は10°間隔・36方位）
- NEDO MONSOLA-11の積雪深10 cm以上の月別出現率を表示
- NEDO PDF帳票の座標・標高・出現率をローカルOCRで読込
- `0.95 − 出現率`による月別発電量係数を計算
- 入力内容をブラウザへ自動保存
- 候補地名、候補地メモ、現地確認メモの入力
- Solar Pro入力用メモのコピー
- チェック結果CSV、地平線CSV（仮形式）、ブラウザの印刷機能を使ったPDF保存

## 起動方法

Node.js 20.19以上（または22.12以上）をインストールしてください。

```bash
npm install
npm run dev
```

表示されたURL（通常は `http://127.0.0.1:5173`）をブラウザで開きます。
`npm run dev` は検証済みの本番ビルドを作成してから表示します。コード変更を即時反映する開発モードを使う場合は `npm run develop` を実行してください。

本番用ファイルは次のコマンドで作成できます。

```bash
npm run build
npm run preview
```

主要な座標・3次メッシュ・NEDO検証ロジックの回帰テストは次で実行できます。

```bash
npm run test
```

同じ社内LANのメンバーへ公開する場合は `start-team-server.cmd` をダブルクリックするか、次を実行します。詳細は [TEAM_SHARING.md](TEAM_SHARING.md) を参照してください。

```bash
npm run share
```

## 別PCへ持ち出して使う場合

GitHubには `node_modules/` と `dist/` は含めていないため、別PCでGitHubからソースだけを取得してもそのままでは起動できません。

もっとも簡単に別PCへ持ち出す場合は、作業PCで `MAKE_PORTABLE_PACKAGE.cmd` をダブルクリックしてください。

作成されるファイル：

```text
outputs/SolarSitePrecheck_v1.1_portable.zip
```

このzipを別PCへコピーして展開し、`RUN_PORTABLE.cmd` をダブルクリックすると、ビルド済みアプリを `http://127.0.0.1:5173/` で起動できます。

別PCにCodex DesktopのNode.jsランタイム、または通常のNode.js 20.19以上が必要です。

## データと注意事項

- 背景地図：国土地理院「全国最新写真（シームレス）」および標準地図
- 住所検索：国土地理院の住所検索サービス
- 標高：NEDO PDF帳票値を優先。帳票読込前は国土地理院DEM標高タイルを補助値として使用します。
- 地平線影響：候補地点から250 m〜5 kmを方位別にサンプリングした簡易指標です。通常は8方位、詳細分析では10°間隔・36方位を取得します。建物・樹木・細かな尾根は反映されず、正式な地平線測量ではありません。
- 積雪：NEDO「年間月別日射量データベース（MONSOLA-11）」の月別積雪深10 cm以上出現率を使用します。最寄り観測地点は参考表示だけとし、係数計算・CSV・レポートには採用しません。候補地点と同じ3次メッシュ番号を確認できたPDF値だけを採用します。
- NEDO PDF読込時は、候補地点から計算したメッシュ番号とPDF内の番号、月別値と年・季節集計値の整合性を検証します。不一致の場合は値を反映しません。
- NEDO PDFは固定位置ではなく表の罫線を検出し、3種類の画像処理で交差読取します。検証履歴と残る制限は `ACCURACY_VALIDATION.md` を参照してください。

画面と出力には出典を表示しています。公開・商用運用へ進む前に、国土地理院コンテンツ利用規約、地理院タイル一覧の個別条件、各外部データの規約・出典表記を改めて確認してください。また、標高取得サービスは将来変更される可能性があるため、本運用ではサーバー側キャッシュや標高タイルの自前解析への移行を推奨します。
