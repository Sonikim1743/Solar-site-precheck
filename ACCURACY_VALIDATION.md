# 精度検証メモ

## 2026-06-25 NEDO PDF検証

- 過去案件フォルダーでNEDO PDFを58件確認し、印刷位置・地域が異なる9件を抽出して反復検証した。
- 固定座標での切り出しを廃止し、最終ページの表罫線（横線・19本の縦線）から積雪行と17列を自動検出する方式へ変更した。
- 各セルを原画像、内側クロップ、二値化の3方式でOCRし、月別12値と年・冬・春・夏・秋の集計値が一致する組合せを採用する。
- 9件すべてでメッシュ番号と積雪17列の整合性検証に合格した。
- 目視で正解を確認した検証用帳票の一例では、`0.81 / 0.94 / 0.56 / 0.02 / 0.00 ... / 0.30` を再現した。

## 採用条件

- 候補地点から計算した3次メッシュ番号がPDF内の番号候補と完全一致する場合だけ積雪値を採用する。
- 最寄り観測地点の値は参考表示だけとし、係数、CSV、レポートへは採用しない。
- PDF標高は国土地理院の候補地点標高を参照してOCR候補を検証し、差が大きい場合は採用しない。
- 過去バージョンで保存された未検証の積雪値・地平線値は自動復元しない。

## 地平線・標高

- 地平線は各方位5点から10点へ増やし、250mから5kmの中間地形を確認する。
- 純粋な地形仰角と、想定樹高を加えた保守的仰角を分けて表示する。
- 取得済みの国土地理院標高点は最大2,000点までブラウザへ保存し、同じ地点の再検証で再利用する。

## 残る制限

- 地平線はDEMによる概算であり、建物、個別樹木、造成後地形は現地写真・測距で補正する。
- 新しい地点の航空写真、住所検索、未取得の標高にはインターネット接続が必要。
- NEDO帳票の様式が将来変更された場合は罫線検出に失敗する可能性があり、その場合は値を採用せずエラー表示する。
