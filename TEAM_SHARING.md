# チーム共有で使う方法

このMVPは、同じ社内LAN・Wi-Fiに接続しているメンバーへブラウザアプリとして公開できます。サーバーとなるPCは、利用中ずっと起動しておいてください。

## 初回準備

1. このフォルダーをサーバーPCへ置きます。
2. `start-team-server.cmd` をダブルクリックします（Codex付属Node.jsがあれば自動利用します）。
3. 黒い画面に表示される `http://192.168.x.x:5173/` をメンバーへ共有します。
4. Windowsファイアウォールが確認を出した場合は、社内のネットワークだけ許可します。

## メンバーが開くURL

サーバーPCでPowerShellを開き、次のコマンドでIPv4アドレスを確認します。

```powershell
ipconfig
```

例えばIPv4アドレスが `192.168.1.25` なら、メンバーは次を開きます。

```text
http://192.168.1.25:5173/
```

サーバーPC自身は `http://127.0.0.1:5173/` で利用できます。

重要：メンバーPCで `127.0.0.1` を開くと、メンバー自身のPCを参照するため接続できません。必ずサーバーPCの `192.168.x.x` アドレスを使ってください。

## 現在の共有範囲

- アプリ本体とMONSOLA-11データは全員で同じものを参照します。
- 入力中の案件データは各メンバーのブラウザに自動保存されます。
- CSV・PDFは各メンバーのPCへ出力されます。
- 現段階では共通の案件一覧、ユーザー管理、同時編集機能はありません。

## 安全上の注意

- このサーバーにはログイン機能がないため、インターネットへ直接公開しないでください。
- 社内LANまたはVPN内だけで使用してください。
- ngrokなどで一時的に社外メンバーへ見せる場合は、短時間だけ公開し、可能な限りBasic認証やIP制限を設定してください。
- ngrokでBasic認証を使う例：

```powershell
ngrok http 5173 --basic-auth "review:共有用パスワード"
```

- 候補地名、メモ、現地確認内容は利用者のブラウザのlocalStorageに保存されます。共有PCや第三者が触れる端末では、検証後にブラウザのサイトデータを削除してください。
- 正式運用では、認証、共通案件DB、バックアップ、HTTPSを追加してください。

## 次の推奨段階

複数人が同じ候補地を保存・検索・更新する必要が出たら、Next.js APIとPostgreSQLまたはSQLiteを追加し、共通案件一覧とユーザー認証を実装します。

候補地調査で共通化する項目は [INPUT_CHECKLIST.md](INPUT_CHECKLIST.md) を参照してください。
